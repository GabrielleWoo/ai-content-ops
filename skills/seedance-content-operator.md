# Seedance Content Operator · AI内容运营操作系统

## 系统定位

这不只是一个内容生成工具，而是一套**可沉淀、可迭代、可开源、可商业化**的 AI 内容运营操作系统。

每次运行产出三层价值：
1. **内容层** — 直接可用的多平台视频发布包
2. **知识层** — 可复用的教程、案例、宣发文案
3. **系统层** — 反馈数据 → Skill 迭代 → 自我进化

## 完整工作流

```
选题输入
  ↓
内容生成（分镜 / 提示词 / 发布包）
  ↓
素材生成（即梦CLI 生图/生视频）
  ↓
教程 & 案例沉淀（可复用 + 可分享）
  ↓
宣发文案生成（推广工作流本身）
  ↓
飞书同步（知识库备份 + 更新日志）
  ↓
GitHub 开源更新（README + 案例 + Skill）
  ↓
发布（抖音 / YouTube / X / 小红书）
  ↓
数据反馈（录入飞书 → 触发迭代）
  ↓
Skill 自进化（复盘结论 → 更新本文件）
```

## 内容层规则

### 默认规格

- YouTube：16:9，12-15 秒（短片）或 4段×15s 拼接（1分钟），标题 + 简介 + 标签 + 置顶评论
- 抖音：9:16，12-15 秒，钩子 + 话题标签 + 竖屏封面
- X：图片或视频推文，工作流复盘角度，简洁文案
- 视觉风格：匹配主题。未指定时城市文旅默认高能电影感

### 分镜脚本规则

每个镜头包含：镜号 / 时间 / 画面 / 动作 / 镜头运动 / 场景光线 / Seedance核心提示词

每个镜头聚焦一件事。

### 参考板规则

一张图包含：主视觉风格 / 角色场景锚点 / 地标道具 / 色彩板 / 6-8个分镜缩略图 / 短标签

### 视频提示词规则

必须包含：画幅 / 时长 / 参考图为严格视觉锚点 / 负面提示词

```
Negative: wrong city, inconsistent landmarks, blurry text, watermark, logo, random subtitles, extra limbs, distorted faces, dull colors, black bars, cropped main subject.
```

## 视觉参考体系 & 素材生成升级流程

> 从「能生成」升级到「生成得好」。先快速验证，再精准补救，用参考图驱动一切。

### 决策树

```
Step 1：GPT-image-2 纯文本直出（场景组图 + 概念效果图）
        ↓
   质量自检（3条）：风格命中？构图清晰？色彩基调对？
        ↓
  ┌─────┴──────┐
效果好         效果不好
  ↓                 ↓
直接进          Step 2：补救参考
Seedance    ┌──────────────────┐
生视频       路径A：Pinterest   路径B：生成MJ提示词
            找参考图（≤3张）    → MJ生成参考图
            └──────────────────┘
                     ↓
            Step 3：参考图利用
     ┌────────────────────────────┐
     方案A：参考图→GPT-image-2    方案B：参考图→
     精修概念图/分镜图→生视频      Seedance全能参考
     └────────────────────────────┘ 直接生成视频
```

### Codex CLI · GPT-image-2 图像生成命令

使用 OpenAI Codex CLI（`codex exec`）调用 GPT-image-2 生成参考板、教程图、宣传图：

```bash
# 生成单张图并保存到本地
codex exec \
  --model gpt-5.5 \
  -s workspace-write \
  "Generate an image [描述] and save it as PNG to [输出路径]"

# 批量生成系列图（在脚本中串行调用）
for prompt in "${PROMPTS[@]}"; do
  codex exec --model gpt-5.5 -s workspace-write "$prompt"
done
```

**参数说明：**
- `--model gpt-5.5` — 使用支持图像生成的模型（gpt-image-2 作为工具自动调用）
- `-s workspace-write` — 允许写入当前目录及 /tmp
- 生成的图片会自动保存到 `~/.codex/generated_images/` 并可被 Codex 复制到指定路径

**注意：** `gpt-image-2` 不能直接作为 `--model` 参数使用（需 API Key），应通过 gpt-5.5 的工具调用触发。

### GPT-image-2 必输出的组图类型

每个场景必须生成：
1. **场景全景概念图** — 16:9，建立空间感和整体氛围
2. **关键道具特写** — 1:1，角色持有物/标志性物件
3. **光线/色彩测试图** — 验证调色板是否命中
4. **分镜缩略图组** — 3-6个关键镜头画面预览

**质量自检（3条全中才算通过）：**
- [ ] 视觉风格与设定吻合（不是"差不多"，是"就是这个感觉"）
- [ ] 主体构图清晰，细节可辨认
- [ ] 色彩基调命中（冷暖/饱和度/对比度符合预期）

### MJ 提示词公式

```
[核心场景描述]，[视觉风格参考]，
[光线描述]，[色彩基调]，
[镜头/构图指令]，[情绪/氛围关键词]，
[技术参数]
--style raw --ar 16:9 --v 6.1 --q 2
```

*电影大场景：*
```
[scene] interior concept art, [style e.g. Pixar/Ghibli],
volumetric [color] light shafts, [palette],
wide establishing shot, cinematic composition, [mood],
hyper-detailed, atmospheric fog, professional concept art
--style raw --ar 16:9 --v 6.1 --q 2
```

*角色互动中景：*
```
[character full description], [action], in [scene],
[lighting], [color palette], medium shot,
expressive facial detail, [mood], character concept art
--style raw --ar 3:4 --v 6.1 --q 2
```

*道具特写：*
```
close up of [prop], [material texture], [lighting],
[style: photorealistic/illustrated/painterly],
product shot composition, high detail, [color accent]
--style raw --ar 1:1 --v 6.1 --q 2
```

### Seedance 全能参考使用规则

```bash
dreamina text2video \
  --model_version seedance2.0 \
  --ratio 16:9 \
  --duration 10 \
  --reference [参考图路径] \
  --reference-weight 0.85 \
  --prompt "保持参考图风格/光线/色彩不变。[角色动作+镜头运动+情绪变化]。" \
  2>&1
```

- 参考权重：`0.7–0.9`（风格强锁定用0.85+，允许发挥用0.7）
- 一次视频只用 **1张** 主参考图
- 提示词侧重描述「动态」，风格让参考图定义

### MJ vs GPT-image-2 选型

| 用途 | 选择 |
|------|------|
| 建立视觉风格基准 / 风格探索 | **MJ** |
| 按需精准生成分镜/概念图 | **GPT-image-2** |
| 已有参考图直接出视频 | **Seedance全能参考** |

**口诀：MJ 定风格 → GPT-image-2 造内容 → Seedance 变视频**

### 每场景执行清单

```
□ 写场景描述（中文设定 → 英文提示词）
□ GPT-image-2 直出（场景组图+概念图）
□ 质量自检（3条标准）
  → 通过 → 进视频生成
  → 不通过：
    □ Pinterest找ref（≤3张）OR MJ提示词→MJ出图
    □ 参考图确认（风格对了吗？）
    □ 方案A：参考图→GPT-image-2精修→视频
      OR
    □ 方案B：参考图→Seedance全能参考→视频
□ 更新飞书案例文档 T2 时间节点
```

## 时间维度追踪

每次运行必须记录并输出时间节点。这是系统可量化、可优化的核心数据层。

### 创作生命周期

```
T0  灵感/选题确定
T1  内容包完成（Skill输出完整内容包的时刻）
T2  素材生成完毕（参考板 + 全部视频ACT完成）
T3  审核就绪（剪辑/校审完成）
T4  发布完成（最后一个平台上线）
T5  数据复盘（发布48h后录入数据+写出迭代结论）
```

### 自动记录规则

- **运行开始**：输出 `🕐 T1 开始：YYYY-MM-DD HH:mm`
- **内容包完成**：输出 `⏱ 内容包耗时：X min`
- **素材提交时**：记录 `dreamina submit_id + 提交时间戳`
- **飞书同步时**：在案例文档时间追踪表更新对应节点
- **案例卡片底部**：必须附时间轴摘要

### 时间轴摘要格式（案例卡片必填）

```
⏱ 时间轴摘要
T0 → T1: Xh Xmin（灵感→内容包）
T1 → T2: Xh Xmin（内容包→素材完成）
T2 → T4: Xh Xmin（素材→发布）
全流程: Xh Xmin
效率评级: ⚡极速(<2h) / ✅标准(<6h) / 🐢待优化(>6h)
```

### 效率基准线（随案例积累更新）

| 流程段 | 目标值 | 当前基准（东京案例） |
|--------|--------|----------------------|
| 灵感→内容包 | <30min | ~2.5h（首案例） |
| 内容包→参考板 | <1h | ~16h（多轮迭代） |
| 参考板→视频完成 | <3h | 进行中 |
| 视频完成→发布 | <30min | 待定 |
| 全流程 | <6h | 待定 |

每完成一个案例后更新此基准线。
